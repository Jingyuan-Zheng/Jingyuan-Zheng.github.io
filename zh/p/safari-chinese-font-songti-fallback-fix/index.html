Safari 中文字体变宋体？用一个 CSS 样式表恢复苹方

Safari 浏览中文网页时，部分文字会从苹方突然回退成宋体或明体。本文介绍一个只修复 CJK 字体回退、不强制替换英文与网页原有衬线设计的轻量 CSS 方案。

文章来源： https://jingyuan-zheng.github.io/zh/p/safari-chinese-font-songti-fallback-fix/

本文介绍 Safari 中文字体变宋体的原因与修复方法：通过一个轻量级用户样式表，让本该使用无衬线字体的中文优先回到苹方，同时保留网站有意使用的宋体、楷体和仿宋。

如果你在 Mac 上主要使用 Safari，可能见过这种很难忽略的画面：同一个网页里，大部分中文是苹方，某个按钮、导航、搜索结果或一小段正文却突然变成宋体；英文和数字看起来正常，旁边的中文却像从另一个网站拼了进来。

更让人困惑的是，同一个页面放到 Chrome 里往往又是正常的。

我为这个问题整理了一个开源解决方案：Safari CJK Font Fallback Fix。项目只有一个 CSS 文件，不需要安装 Safari 扩展，也不包含 JavaScript。

项目地址：Jingyuan-Zheng/Safari-CJK-Font-Fallback-Fix

这不是字号或编码问题

Safari 中的“字体突然变宋体”，通常不是网页乱码，也不是字号、缩放比例或文字编码出了问题。

很多网站的字体列表首先写的是 Arial、Helvetica、Inter、Roboto 等西文字体。它们可以显示英文和数字，却不一定包含完整的中文字形。浏览器遇到中文时，只能继续沿着字体列表寻找可用字形；如果网站后面又放了宋体，或者沿用了面向 Windows 的微软雅黑、等线、中易黑体等字体名，Safari 最终选到的中文字体就可能与网站想要的无衬线风格不一致。

这就是字体回退（font fallback）：前面的字体没有这个字符，浏览器便寻找下一种能画出它的字体。

不同浏览器处理字体名称和回退顺序的方式并不完全相同，所以才会出现“Chrome 是苹方，Safari 却变成宋体”的情况。WebKit 对语言与 CJK 字体选择的讨论已经持续多年；Safari 目前也仍然保留了导入用户样式表的入口，Apple 在官方说明中将它放在“Safari 设置 → 高级 → 样式表”。

为什么常见的全局替换并不理想

网上最常见的解决办法，是给所有元素强制设置一种字体：

1
2
3

* {
    font-family: "PingFang SC" !important;
}

它确实简单，也经常能立刻让宋体消失，但代价是把整个网页的字体设计一起覆盖掉。

图标字体可能因此显示成方块，代码块失去等宽字体，英文标题不再使用网站原来的字体；更重要的是，新闻正文、古籍内容或品牌页面本来就可能有意选择宋体。把所有字体都替换为苹方，并不是修复回退，而是重新设计整个网站。

在 V2EX 关于 Safari 自定义字体的讨论中，就有用户遇到全局替换后投票按钮图标变成方格的问题。这个反馈也是我没有采用“全局强制苹方”方案的原因。

这个项目怎样只修复中文部分

这个样式表的核心不是修改网页上的每个元素，而是用 @font-face 和 unicode-range 补全常见西文字体缺失的 CJK 字符。

以 Arial 为例，规则只在遇到汉字、CJK 标点和全角字符时调用苹方：

1
2
3

6
7
8

@font-face {
    font-family: "Arial";
    src: local("PingFangSC-Regular");
    font-weight: 100 599;
    unicode-range: U+2E80-2FFF, U+3000-303F,
        U+3400-4DBF, U+4E00-9FFF, U+F900-FAFF,
        U+FF00-FFEF, U+20000-2FA1F;
}

这样一来，英文字母和数字仍然由 Arial 显示，中文缺口则由 PingFang SC 补上。页面原来的英文宽度、间距和视觉风格不会被一并改掉。

项目还针对 Arial、Helvetica、Verdana、Tahoma、Segoe UI、Roboto、Inter、Open Sans 等常见字体分别补充了中文范围，并把微软雅黑、等线等 Windows 无衬线字体名映射到 macOS 的苹方。繁体中文字体名则对应 PingFang TC。

常规字重和粗体也分别映射到 PingFangSC-Regular 与 PingFangSC-Semibold，避免正文恢复苹方之后，粗体又突然跳回另一套字体。

宋体、楷体和仿宋并没有被禁止。网站明确指定 SimSun、KaiTi、FangSong、PMingLiU 等字体时，样式表会继续把它们对应到 macOS 上合适的宋体、楷体或仿宋字体。

安装方法

整个安装过程只需要一个 CSS 文件：

  1. 下载项目中的 safari-fix.css。
  2. 打开 Safari，进入“Safari 浏览器 → 设置 → 高级”。
  3. 在“样式表”菜单中选择刚刚下载的 safari-fix.css。
  4. 使用 Command+Q 完整退出 Safari，然后重新打开。

只关闭当前标签页或刷新网页，Safari 可能仍然使用之前读取的样式表。修改 CSS 后也建议完整退出并重开 Safari。

如果想停用它，只需回到同一个“样式表”菜单，改回“未选择”即可，不会修改系统字体或网站文件。

修复后什么会改变

启用后，原本应当采用无衬线风格、却因为回退错误而显示成宋体的中文，会优先使用 PingFang SC；繁体中文无衬线字体名称会对应到 PingFang TC。

与此同时：

  • 英文和数字继续使用网站原本指定的西文字体；
  • 图标字体和代码字体不会因为全局 font-family 覆盖而被统一替换；
  • 网站明确选择的宋体、楷体和仿宋仍然保留；
  • 普通与粗体中文会使用匹配的苹方字重；
  • 不需要为每一个网站分别安装脚本或扩展。

它修复的是“意外回退”，不是把整个互联网统一成一种字体。

隐私与适用范围

项目只有 CSS，不含 JavaScript，不读取浏览记录、账号或网页内容，不包含统计与遥测，也不会发起网络请求。

它面向 macOS 上的 Safari。跨域 iframe、Shadow DOM、Canvas 绘制的文字，以及图片中的文字可能不受用户样式表影响。网站如果刻意采用“英文无衬线、中文宋体”的混排设计，启用后其中的中文也可能改用苹方；CSS 无法百分之百猜中每个设计者的意图。

常见问题

为什么导入后没有立即生效？

请使用 Command+Q 完整退出 Safari，再重新打开。普通刷新通常不会重新读取用户样式表。

它会把所有宋体都换成苹方吗？

不会。项目保留了明确指定的宋体、楷体和仿宋，只处理本应采用无衬线字体却发生错误回退的情况。

会影响英文 Arial 或 Helvetica 吗？

不会。unicode-range 只为 CJK 字符补充苹方，拉丁字母、数字、字宽和间距仍由原字体处理。

需要安装字体或 Safari 扩展吗？

不需要。苹方是 macOS 的系统中文字体，Safari 本身支持加载用户样式表。

一个小问题，也值得被认真修好

字体回退很少让网页彻底不能用，但它会不断制造细小的视觉跳变。尤其在一整页干净的苹方中突然出现一行宋体时，你很难再假装没有看见。

这个项目不试图替网站决定所有排版，只在浏览器缺少合适的中文无衬线字形时，补上更符合 macOS 阅读体验的那一块。一个 CSS 文件，随时启用、随时关闭，也不要求把浏览权限交给扩展。

如果你遇到新的 Safari 中文字体回退问题，可以在 GitHub Issues 提交公开复现网址、实际 font-family、macOS 与 Safari 版本，以及预期和实际字体。请不要附上私人网址、账号信息或浏览记录。
